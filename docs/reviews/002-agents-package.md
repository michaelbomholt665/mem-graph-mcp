## Suggestions

| # | File | Line | Suggestion | Category |
|---|------|------|------------|----------|
| 1 | `orchestrator_graph.py` | ~285 | **Data race on shared `all_patches` dict in concurrent workers.** Inside `LogicDraftNode.run`, multiple coroutines run concurrently via `anyio.create_task_group` and both call `all_patches.update(...)`. CPython's GIL protects individual dict operations, but the intent is parallel async execution and future non-GIL builds (CPython 3.13+) will break this. Use an `asyncio.Lock` or collect results per-worker and merge after the task group exits. | Correctness |
| 2 | `orchestrator_graph.py` | ~380 | **`import anyio` inside run method.** The `anyio` import is repeated inside `LogicDraftNode.run()` and `StyleDraftNode.run()` as function-level imports. Python caches module imports, so there is no real cost, but it is inconsistent with the module-level imports elsewhere and hides the dependency. Move to module top. | Maintainability |
| 3 | `document/triage_agent.py` | ~65 | **`line_start: int = Field(default=0)` uses 0 to mean "unknown" on 1-indexed lines.** Downstream consumers that zero-check for truthiness (`if finding.line_start`) will silently miss line 0 which is meaningless anyway; those that display the number will show `0` to users. Use `Optional[int] = None` to express "unknown". | Correctness |
| 4 | `orchestrator_agent.py` | ~265 | **`_invoke_subagent` raises `ValueError` for unknown sub-agent names.** This is inside a broad `except Exception` handler that converts it to a `BatchResult(failed=True)`. An unknown `subagent_name` will silently produce a failed batch for every batch rather than failing fast at configuration time. Add an explicit allowlist check when the `OrchestratorDependencies` are constructed. | Correctness |
| 5 | `fix/fixer_agent.py` | whole file | **LLM-generated `proposed_snippet` applied to files without diff verify.** The `FilePatch.original_snippet` / `proposed_snippet` fields are raw strings generated by the LLM. The downstream tool that applies them should verify the `original_snippet` actually appears in the target file before replacing it, to prevent hallucinated patches silently corrupting files. | Security |
| 6 | `audit/audit_agent.py` | ~319 | **`file_extension` from deps used directly in glob pattern.** A dep value like `".py; rm -rf /"` would pass through `glob.glob` safely (it just won't match), but it would also pass in the system prompt. Sanitise to an allowlist of known extensions. | Security |
| 7 | `router_agent.py` | ~155 | **Context violations truncated to 20, decisions to 10 in system prompt.** This is a reasonable guard against context overflow, but the truncation is silent â€” the agent has no way to know more context was omitted. Add a note like `"(showing first 20 of N total)"` when the list is truncated so the agent can report incomplete context. | Correctness |

## What Looks Good

- **Pydantic AI dependency injection** â€” All agents use `@dataclass` `Deps` objects cleanly; no global state leaks between agent runs.
- **`defer_model_check=True`** â€” Correctly applied to all agents, preventing import-time provider validation failures in tests.
- **`_MAX_FILE_BYTES = 64_000`** â€” Both `audit_agent` and `orchestrator_agent` enforce this in their read helpers. File truncation is flagged in the output.
- **`orchestrator_graph.py` pydantic-graph six-node DAG** â€” The Contextâ†’Sentryâ†’Logicâ†’Styleâ†’Guardâ†’Sync lifecycle is clean and the retry loop with `max_retries` correctly prevents infinite loops.
- **`AuditDependencies` rule injection** â€” Passing `rules: list[AuditRule]` makes the audit agent fully composable for domain-specific auditing without subclassing.
- **`TriagedViolation.existing_violation_id`** â€” Explicit recurrence tracking linking triage decisions to graph violations is a strong deduplication design.
- **`RouterDecision.solo_mode`** â€” The solo-mode bypass for high-complexity/large tasks avoids batching overhead when full context is needed.
