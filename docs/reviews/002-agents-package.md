# Code Review: `src/mem_graph/agents/`

**Reviewed:** 2026-04-14
**Scope:** `orchestrator_agent.py`, `orchestrator_graph.py`, `router_agent.py`, `audit/audit_agent.py`, `audit/rule_injector_agent.py`, `document/decision_agent.py`, `document/triage_agent.py`, `document/task_agent.py`, `document/scribe_agent.py`, `fix/fixer_agent.py`, `map/map_agent.py`, `validate/validation_agent.py`, `validate/sentry_agent.py`

---

## Summary

The agents package implements a layered, Pydantic AI‚Äìbased multi-agent system. The persona injection, structured outputs, and test-first `SentryNode` pattern are well-designed. Two issues are critical: an SSRF vulnerability in the Rule Injector tool and an unvalidated path-traversal risk in the `list_files` tool. There is also a data-race risk in the parallel Fixer/Scribe workers inside `orchestrator_graph.py`.

---

## Critical Issues

| # | File | Line | Issue | Severity |
|---|------|------|-------|----------|
| 1 | `audit/rule_injector_agent.py` | ~160 | **SSRF ‚Äî `endpoint` parameter accepted from the LLM.** The tool `rule_injector_fetch_external_rules(ctx, endpoint: str)` performs an HTTP GET to the `endpoint` argument supplied by the agent at runtime. The guard `if not ctx.deps.external_api_url: return...` only checks whether the feature is enabled ‚Äî it does **not** validate the `endpoint` value. A prompt-injection or jailbroken model can call this tool with `endpoint="http://169.254.169.254/latest/meta-data/"` (or any internal address). Fix: ignore the `endpoint` parameter entirely and use `ctx.deps.external_api_url` for all HTTP requests. | üî¥ Critical |
| 2 | `orchestrator_agent.py` / `audit/audit_agent.py` | ~194 / ~319 | **Path traversal in `list_files` tool.** Both `list_files` tools construct `glob.glob(os.path.join(ctx.deps.package_path, f"**/*{ext}"), recursive=True)` without validating that `package_path` is contained within an allowed root. A caller who controls `package_path` (or `file_extension`) can enumerate arbitrary filesystem paths. Fix: resolve the path with `Path(package_path).resolve()` and assert it is inside the project root before globbing. | üî¥ Critical |

## Suggestions

| # | File | Line | Suggestion | Category |
|---|------|------|------------|----------|
| 1 | `orchestrator_graph.py` | ~285 | **Data race on shared `all_patches` dict in concurrent workers.** Inside `LogicDraftNode.run`, multiple coroutines run concurrently via `anyio.create_task_group` and both call `all_patches.update(...)`. CPython's GIL protects individual dict operations, but the intent is parallel async execution and future non-GIL builds (CPython 3.13+) will break this. Use an `asyncio.Lock` or collect results per-worker and merge after the task group exits. | Correctness |
| 2 | `orchestrator_agent.py` | ~560 | **`_get_state` stores mutable state on the `RunContext` object via dynamic attribute.** `ctx._orch_state = {...}` is undocumented usage of `RunContext`. If Pydantic AI re-creates the context between tool calls this state will be reset silently, losing accumulated batch results. Prefer an explicit state container injected via `OrchestratorDependencies`. | Correctness |
| 3 | `orchestrator_graph.py` | ~380 | **`import anyio` inside run method.** The `anyio` import is repeated inside `LogicDraftNode.run()` and `StyleDraftNode.run()` as function-level imports. Python caches module imports, so there is no real cost, but it is inconsistent with the module-level imports elsewhere and hides the dependency. Move to module top. | Maintainability |
| 4 | `document/triage_agent.py` | ~65 | **`line_start: int = Field(default=0)` uses 0 to mean "unknown" on 1-indexed lines.** Downstream consumers that zero-check for truthiness (`if finding.line_start`) will silently miss line 0 which is meaningless anyway; those that display the number will show `0` to users. Use `Optional[int] = None` to express "unknown". | Correctness |
| 5 | `orchestrator_agent.py` | ~265 | **`_invoke_subagent` raises `ValueError` for unknown sub-agent names.** This is inside a broad `except Exception` handler that converts it to a `BatchResult(failed=True)`. An unknown `subagent_name` will silently produce a failed batch for every batch rather than failing fast at configuration time. Add an explicit allowlist check when the `OrchestratorDependencies` are constructed. | Correctness |
| 6 | `fix/fixer_agent.py` | whole file | **LLM-generated `proposed_snippet` applied to files without diff verify.** The `FilePatch.original_snippet` / `proposed_snippet` fields are raw strings generated by the LLM. The downstream tool that applies them should verify the `original_snippet` actually appears in the target file before replacing it, to prevent hallucinated patches silently corrupting files. | Security |
| 7 | `audit/audit_agent.py` | ~319 | **`file_extension` from deps used directly in glob pattern.** A dep value like `".py; rm -rf /"` would pass through `glob.glob` safely (it just won't match), but it would also pass in the system prompt. Sanitise to an allowlist of known extensions. | Security |
| 8 | `router_agent.py` | ~155 | **Context violations truncated to 20, decisions to 10 in system prompt.** This is a reasonable guard against context overflow, but the truncation is silent ‚Äî the agent has no way to know more context was omitted. Add a note like `"(showing first 20 of N total)"` when the list is truncated so the agent can report incomplete context. | Correctness |

## What Looks Good

- **Pydantic AI dependency injection** ‚Äî All agents use `@dataclass` `Deps` objects cleanly; no global state leaks between agent runs.
- **`defer_model_check=True`** ‚Äî Correctly applied to all agents, preventing import-time provider validation failures in tests.
- **`_MAX_FILE_BYTES = 64_000`** ‚Äî Both `audit_agent` and `orchestrator_agent` enforce this in their read helpers. File truncation is flagged in the output.
- **`orchestrator_graph.py` pydantic-graph six-node DAG** ‚Äî The Context‚ÜíSentry‚ÜíLogic‚ÜíStyle‚ÜíGuard‚ÜíSync lifecycle is clean and the retry loop with `max_retries` correctly prevents infinite loops.
- **`AuditDependencies` rule injection** ‚Äî Passing `rules: list[AuditRule]` makes the audit agent fully composable for domain-specific auditing without subclassing.
- **`TriagedViolation.existing_violation_id`** ‚Äî Explicit recurrence tracking linking triage decisions to graph violations is a strong deduplication design.
- **`RouterDecision.solo_mode`** ‚Äî The solo-mode bypass for high-complexity/large tasks avoids batching overhead when full context is needed.

---

## Verdict

**Request Changes** ‚Äî The SSRF in `rule_injector_fetch_external_rules` and path traversal in `list_files` are critical and must be remediated. The parallel write race and stateful `RunContext` mutation are high-priority correctness bugs.
